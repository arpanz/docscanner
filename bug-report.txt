COMPREHENSIVE BUG & ISSUE REPORT
🔴 CRITICAL/FUNDAMENTAL UX ISSUES
1. Camera Page Doesn't Actually Show Camera (Major UX Failure)
The camera page shows only a loading spinner and immediately launches the native flutter_doc_scanner which takes over the entire screen. Users never see an actual camera preview within your app.
2. No Crop/Edge Detection Control
The flutter_doc_scanner package handles everything internally. Your CropEnhanceSheet exists but is never used anywhere in the codebase. Users cannot manually adjust crop corners or choose enhancement filters during capture.
3. Adding Pages to PDF Documents Silently Fails
When user tries to add pages to an existing PDF document via the "Add pages" button, it just pops back with no explanation. Should either support appending PDF pages OR show a clear error message.
4. Title Prompt Blocks Workflow
After scanning, users MUST enter a title before seeing their document. If they cancel, the entire scan is lost. Should auto-generate a title and allow editing later.
🟠 LOGICAL ERRORS
5. Export Sheet Doesn't Handle PDF Documents
When exporting a PDF-based document, it still tries to build a new PDF from image paths. But PDF documents store the PDF path directly in imagePath, not individual page images.
6. PageItem Widget Crashes on PDF Files
Tries to load PDF files as images using Image.file(). Will throw an error when viewing PDF-based documents since PDFs can't be displayed with Image.file().
7. Reorder Mode Shows Wrong UI for PDFs
The reorder mode assumes all pages are images and uses Image.file() for thumbnails. Won't work for PDF documents.
8. Delete Page Logic Broken for PDFs
Allows deleting pages even for single-page PDF documents. A PDF document has exactly one "page" (the whole PDF). Deleting it should delete the entire document.
9. Gallery Import Has No Image Processing
Images picked from gallery are used directly without any cropping, enhancement, or compression quality control. Users expect the same processing as camera scans.
10. Share/Duplicate/Export Features Are Placeholders
Long-press menu shows "coming soon" for Share, Export PDF, and Duplicate - but these features actually exist elsewhere in the app. This is confusing and inconsistent.
🟡 UI/UX ERRORS
11. No Visual Feedback During Scan Processing
Only shows a spinner while the native scanner launches. No feedback during actual scanning, processing, or saving.
12. Empty State Misleading
Empty state says "Tap the button below to scan" but the FAB is centered at bottom, not "below" the empty state content.
13. Page Counter Overlaps Content
The page counter pill may overlap with the delete button in PageItem on smaller screens.
14. No Haptic Feedback
No haptic feedback on capture, delete, or other important actions. Standard in modern mobile apps.
15. Settings Page Is Non-Functional
All settings options have empty onTap: () {} handlers. Theme toggle, cache clearing do nothing.
16. Favourite Feature Half-Implemented
Database has isFavourite column and DAO has toggleFavourite method, but there's NO UI anywhere to mark documents as favourites or filter by favourites.
🟢 CODE QUALITY & POTENTIAL BUGS
17. Race Condition in Image Capture
In _addPages, if some images fail and some succeed, successCount is tracked but pageIndex uses startIndex + successCount which could create gaps or duplicates if failures happen mid-batch.
18. Missing Null Safety Check
Double negation logic is hard to follow and error-prone.
19. Memory Leak Risk
If the widget rebuilds with a different docId, the controller isn't reset. Could cause scroll position issues.
20. No Error Handling for Missing Files
If a page file is deleted externally or fails to load, the app shows a small error text but doesn't offer to remove the broken page or recover.
21. OCR Feature Completely Missing
OCR dependency is included but nowhere used. Database has ocrText column but it's never populated or searched.
22. Search Doesn't Search OCR Text
Search only looks at document titles, not the ocrText field that was designed for this purpose.
23. Sort Option "Most Pages" Is Confusing
Sorting by page count descending isn't very useful for most users. More useful sorts would be: recently viewed, favourites first, size.
24. No Document Size Display
Users can't see how large a document is (in MB). Important for storage management and sharing.
25. Migration May Fail
Migration adds ocrText column but doesn't handle the case where the database might be corrupted or from a much older version.
26. No Undo After Delete
Deleting a document or page is permanent with no undo option. Standard pattern is to show a snackbar with "Undo" for a few seconds.
27. Inconsistent Dialog Styles
Some dialogs use AlertDialog, some use custom sheets. Button styles and ordering are inconsistent.
28. Export Progress Not Shown
Shows a loading spinner but no progress indication for multi-page documents. Large exports feel frozen.
29. No Multi-Select for Batch Operations
Can't select multiple documents to delete, share, or export together. Tedious for bulk operations.
30. Router Query Parameter Bug
Camera route parses docId from query params but the camera page's PDF handling immediately exits, making this parameter mostly useless.
